Nivel Basico — Error puntual en forecasting

Una cadena retail peruana predice sus ventas semanales en S/. 40 semanas de entrenamiento, 12 de evaluacion. Las metricas son las mismas que en regresion, pero con matices importantes por el orden temporal.

MAE — error promedio en S/. RMSE — penaliza semanas criticas MAPE — error porcentual semanal
Serie de ventas — 52 semanas (40 train + 12 test)
Train (real) Test (real) Prophet
Series de tiempo vs regresion — que cambia
ORDEN TEMPORAL IMPORTA
El set de test siempre es el futuro. No puedes barajar los datos. El error en la semana 45 depende de lo que el modelo aprendio en semanas 1-40.
EL ERROR CRECE CON EL HORIZONTE
Predecir la semana siguiente es mucho mas facil que predecir 12 semanas adelante. Las metricas globales ocultan este deterioro.
ESTACIONALIDAD Y TENDENCIA
Un modelo puede tener buen MAPE global pero fallar en semanas pico. Los errores tienen estructura temporal, no son aleatorios.

MAE
Error Absoluto Medio — S/. por semana
S/. 272
⌄
FORMULA
MAE = (1/h) ∑ |y‑t − ŷ‑t|
h = horizonte de prediccion (12 semanas). Se calcula igual que en regresion pero solo sobre el test set, respetando el orden temporal. S/.272 = desviacion promedio por semana respecto a ventas reales.
Matiz en series de tiempo

El MAE global puede enmascarar que el error en las primeras semanas (horizonte corto) es mucho menor que en las ultimas (horizonte largo).

RMSE
Raiz del Error Cuadratico Medio — penaliza semanas criticas
S/. 423
⌄
FORMULA
RMSE = √[(1/h) ∑ (y‑t − ŷ‑t)²]
En retail, una semana con error muy alto (campana, evento) puede desestabilizar el inventario. El RMSE penaliza esos picos mas que el MAE.

Ratio RMSE/MAE = 423/272 = 1.55: hay 1-2 semanas con errores significativamente mayores al promedio.
Cuando RMSE/MAE me preocupa

Ratio cercano a 1 → errores homogeneos, el modelo falla uniformemente.

Ratio > 1.5 → hay semanas con errores grandes. Investiga que ocurrio: ¿promocion no registrada? ¿quiebre de stock? ¿feriado?

Accion: agregar variable de evento al modelo para capturar esos picos.
MAPE
Error Porcentual Absoluto Medio — comparable entre tiendas
3.8%
⌄
FORMULA
MAPE = (100/h) ∑ |y‑t − ŷ‑t| / |y‑t|
Permite comparar modelos entre tiendas con distinto nivel de ventas. Una tienda de S/.3,000/sem y otra de S/.15,000/sem tendran MAPE comparables aunque sus MAE sean muy distintos.
MAPE < 5%Excelente ← aqui
5% – 10%Bueno
10% – 20%Aceptable
> 20%Revisar modelo
Limitacion critica en series de tiempo

Si hay semanas con ventas cero o muy bajas (feriado, cierre), el MAPE explota a infinito porque divides entre cero.

Solucion: excluir esos periodos, usar SMAPE, o usar MASE (ver Nivel Intermedio).

Ademas, el MAPE penaliza asimetricamente: sobrestock y rotura de stock dan el mismo MAPE aunque el impacto en negocio sea muy distinto.
💡 En forecasting de retail, MAPE < 10% se considera operativamente util para planificacion de inventario.

Nivel Intermedio — MASE, SMAPE y error por horizonte

El MAPE tiene problemas con ceros y es asimetrico. El MASE lo resuelve comparando contra un modelo naive. El error por horizonte revela si el modelo se degrada al predecir mas lejos en el tiempo.

MASE — vs predictor naive SMAPE — error % simetrico Error por horizonte
MASE
Mean Absolute Scaled Error — vs predictor naive
1.08
⌄
FORMULA
MASE = MAE‑modelo / MAE‑naive
El predictor naive predice que el valor de manana sera igual al de hoy: ŷ‑t = y‑(t-1).

MAE naive = S/.252
MAE Prophet = S/.272
MASE = 272/252 = 1.08
MASE < 1Mejor que naive 🔥
MASE = 1Igual que naive
MASE > 1Peor que naive 🙈
MASE = 1.08: lectura honesta

Prophet apenas supera en 8% al predictor mas simple posible. No es necesariamente malo: en series con alta variabilidad, el naive es dificil de vencer.

Por que MASE y no MAPE?
1. No explota con ventas cero
2. Escala interpretable (referencia = 1)
3. Es simetrico
4. Estandar en competencias M1–M4 de forecasting
💡 Si tu modelo tiene MASE > 1 en produccion, la semana anterior lo hubiera hecho mejor. Es la metrica de validacion mas honesta.
SMAPE
Symmetric MAPE — penaliza sobreestimacion y subestimacion igual
3.6%
⌄
FORMULA
SMAPE = (200/h)∑|y‑t−ŷ‑t|/(|y‑t|+|ŷ‑t|)
Divide entre la suma del real y el predicho. Resultado: sobreestimar 50% y subestimar 50% dan el mismo SMAPE. Rango: 0% a 200%.

SMAPE (3.6%) < MAPE (3.8%): el modelo tiende ligeramente a sobreestimar.
SMAPE vs MAPE en retail

Sobreestimar ventas → sobrestock, capital inmovilizado.
Subestimar ventas → rotura de stock, ventas perdidas.

El MAPE penaliza estas dos situaciones asimetricamente. El SMAPE las trata igual. Antes de elegir, define si el costo de sobrestock y rotura de stock son equivalentes en tu negocio.
Error por horizonte — como se degrada la prediccion

Las primeras semanas son mas faciles de predecir. El error crece a medida que el modelo predice mas lejos en el futuro.

💡 Si el error crece mucho: considera forecasting recursivo (usa h=1 como input de h=2) o un ensemble de modelos distintos para distintos horizontes.

Nivel Avanzado — Intervalos de prediccion y comparador

Una prediccion puntual dice "venderemos S/.7,500". Un intervalo dice "venderemos entre S/.6,800 y S/.8,200 con 95% de probabilidad". La cobertura mide si ese intervalo realmente captura los valores reales.

Intervalos 80% y 95% Coverage — calibracion del intervalo ARIMA vs Prophet vs ETS
Forecast con intervalos de prediccion
Real Prediccion IC 80% IC 95%
Coverage — calibracion de los intervalos

Si dices "intervalo de 95%", el 95% de los valores reales deberia caer dentro. Si solo el 60% cae dentro, el intervalo esta mal calibrado (demasiado estrecho).

Como interpretar:
• Cobertura ≈ nivel nominal → bien calibrado
• Cobertura > nominal → intervalo demasiado ancho (conservador)
• Cobertura < nominal → intervalo demasiado estrecho (subestima incertidumbre)
