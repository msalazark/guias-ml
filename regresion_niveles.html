Nivel Basico — Las 3 metricas esenciales

Tu modelo predice el revenue diario en S/. de 20 tiendas e-commerce. Para saber si funciona bien necesitas responder: ¿cuanto me equivoco en soles? ¿me afectan los errores grandes? ¿cuanto explica el modelo?

MAE — error promedio en S/. RMSE — penaliza errores grandes R² — poder explicativo
MAE
Error Absoluto Medio
S/. 412
⌄
FORMULA
MAE = mean( |y − ŷ| )
Promedia los errores absolutos. Si el modelo predijo S/. 1,390 para una tienda que vendio S/. 1,200, el error es |1200−1390| = S/. 190. Suma esos errores de todas las tiendas y saca la media.
Como interpreto S/. 412?

En promedio el modelo se equivoca S/. 412 por tienda por dia. Es el numero que le muestras al negocio: directo, en la misma unidad del revenue.

Es bueno? Si las tiendas venden ~S/. 4,000/dia, un MAE de S/. 412 es ~10% — aceptable para empezar.
💡 Cuando priorizarlo: cuando todos los errores (grandes y pequenos) importan igual. Util en planificacion de stock.
RMSE
Raiz del Error Cuadratico Medio
S/. 536
⌄
FORMULA
RMSE = √mean( (y − ŷ)² )
Eleva cada error al cuadrado antes de promediar. Penaliza mucho mas los errores grandes. Un error de S/. 2,000 contribuye 4x mas que uno de S/. 1,000.
RMSE VS MAE — SENAL CLAVE
RMSE (S/.536) > MAE (S/.412) en S/.124. Esa diferencia indica tiendas con errores muy grandes que jalan el RMSE hacia arriba.
Como interpreto S/. 536?

El "error tipico" ponderando mas los errores grandes. Misma unidad que el MAE, se comunica igual al negocio.

Si RMSE ≈ MAE → errores homogeneos.
Si RMSE » MAE → hay predicciones muy malas en algunas tiendas.
💡 Cuando priorizarlo: cuando los errores grandes tienen consecuencias serias (rotura de stock, sobrecompra de inventario).
R²
Coeficiente de Determinacion
0.78
⌄
FORMULA
R² = 1 − SS_res / SS_tot
Que proporcion de la variacion del revenue explica el modelo. R²=1 es prediccion perfecta. R²=0 es tan bueno como predecir la media siempre.
R² < 0.3 — Modelo muy debil
R² 0.3–0.6 — Aceptable baseline
R² 0.6–0.85 — Bueno ← nuestro caso
R² > 0.85 — Excelente (verifica leakage)
Como interpreto R² = 0.78?

El modelo explica el 78% de la variacion en ventas. El 22% restante son factores no capturados: promociones, eventos, estacionalidad.

Limitacion critica: R² siempre sube al agregar variables, aunque sean ruido puro. Por eso en Intermedio veras R² ajustado.
⚠️ Trampa comun: R² alto en train no garantiza buen modelo. Siempre evalua en datos de test.

Nivel Intermedio — Error porcentual y ajuste fino

Ya sabes cuantos soles te equivocas. Ahora: ¿ese error es proporcional al tamano de la tienda? ¿Como corrijo la limitacion del R²? ¿Cuando el MAPE falla?

MSE — base matematica MAPE — error % SMAPE — version robusta R² ajustado — penaliza complejidad
MSE
Error Cuadratico Medio
287,440
⌄
FORMULA
MSE = mean( (y − ŷ)² )
Promedio de los errores al cuadrado. Es la base del RMSE (RMSE = √MSE). Su unidad es S/.² — dificil de interpretar directamente — pero es esencial para la optimizacion matematica de modelos.
Por que existe si RMSE es mas facil?

Los algoritmos de ML minimizan el MSE internamente porque es diferenciable. Al reportar resultados al negocio usas RMSE, pero al entrenar el modelo usas MSE.

MSE = 287,440 → RMSE = √287,440 = S/.536
MAPE
Error Porcentual Absoluto Medio
14.8%
⌄
FORMULA
MAPE = mean(|y−ŷ| / |y|) × 100
Expresa el error como porcentaje del valor real. Un error de S/.500 en una tienda de S/.1,000 (50%) es mucho peor que el mismo error en una de S/.10,000 (5%).
MAPE < 10%Excelente
10% – 20%Bueno ← aqui estamos
20% – 50%Aceptable
> 50%Revisar modelo
Limitacion critica del MAPE

Si una tienda vendio S/.0 un dia (cerrada, feriado), el MAPE explota a infinito porque divides entre cero.

Ademas no es simetrico: un error de +100% (predijiste el doble) pesa igual que −50% aunque la diferencia absoluta sea distinta. Por eso existe el SMAPE.
💡 Usa MAPE cuando ninguna tienda tiene ventas cercanas a cero y quieres comunicar el error en porcentaje.
SMAPE
Error Porcentual Absoluto Simetrico
13.9%
⌄
FORMULA
SMAPE = mean( 2|y−ŷ| / (|y|+|ŷ|) ) × 100
Divide entre la suma del valor real y el predicho, no solo entre el real. Eso lo hace simetrico y mas estable cuando y ≈ 0.
SMAPE (13.9%) vs MAPE (14.8%)

Cuando SMAPE < MAPE, el modelo tiende a sobreestimar en algunas tiendas.

En forecasting de demanda (Amazon, Walmart, Falabella) se usa SMAPE como metrica estandar. Para e-commerce con tiendas que tienen dias de ventas bajas, es mas confiable que el MAPE.
R² adj
R² Ajustado — penaliza variables inutiles
0.74
⌄
FORMULA
R²adj = 1 − (1−R²)(n−1)/(n−k−1)
k = numero de features, n = observaciones. Si agregas una variable inutil, R² sube ligeramente pero R² adj baja porque penaliza el k adicional.
EJEMPLO — NUESTRO MODELO
R² (3 features)0.78
R² adj (3 features)0.74
Diferencia−0.04 (normal)
Cuando me preocupo?

R²=0.85 y R²adj=0.62 → diferencia de 0.23 indica overfitting o variables irrelevantes.

Regla practica:
• Diferencia < 0.05 → normal
• Diferencia 0.05–0.10 → revisar features
• Diferencia > 0.10 → overfitting probable
💡 Siempre reporta R² ajustado cuando comparas modelos con distinto numero de features.
Por que el MAPE importa mas que el MAE en tiendas de distinto tamano

Nivel Avanzado — Seleccion de modelo y diagnostico

Ya puedes medir el error. Ahora el reto es elegir entre modelos que compiten y diagnosticar si el modelo cumple los supuestos estadisticos.

AIC — info penalizada BIC — penaliza mas complejidad Residuales — diagnostico visual Homocedasticidad · Normalidad
AIC
Criterio de Informacion de Akaike
312.4
⌄
FORMULA
AIC = 2k − 2·ln(L̂)
k = numero de parametros. L̂ = log-verosimilitud maxima. El AIC premia el ajuste (mayor L̂ = menor AIC) y penaliza la complejidad (mayor k = mayor AIC). Menor AIC = mejor modelo.
Cuando usar AIC vs BIC?

AIC prefiere modelos mas complejos (mejor poder predictivo). BIC penaliza mas agresivamente y prefiere modelos simples (mejor interpretabilidad).

No tienen interpretacion absoluta. Solo se usan para comparar modelos en los mismos datos. Una diferencia ΔAIC > 10 es decisiva.
BIC
Criterio de Informacion Bayesiano
328.7
⌄
FORMULA
BIC = k·ln(n) − 2·ln(L̂)
La diferencia con AIC: la penalizacion es k·ln(n) en vez de 2k. Con n=20, ln(20)≈3, por lo que el BIC penaliza cada parametro ~1.5x mas que el AIC. Con n grande, el BIC es mucho mas conservador.
Interpretacion de ΔBIC

ΔBIC 0–2 → diferencia no significativa
ΔBIC 2–6 → evidencia moderada
ΔBIC > 10 → evidencia muy fuerte

AIC y BIC pueden contradecirse: uno favorece el modelo complejo, el otro el simple.
